# 小红书订单验证工具

## 项目简介

为企业微信私域运营设计的订单验证工具。用户添加企微后，自动收到验证链接，输入小红书订单号验证，成功后跳转到优惠券页面，同时核销订单号防止重复使用。

## 核心功能

✅ **订单验证**：实时验证小红书订单号是否在数据库中  
✅ **自动核销**：验证成功后立即从数据库删除该订单号  
✅ **智能跳转**：验证成功自动跳转到有赞优惠券页面  
✅ **错误提示**：订单号不存在或已使用时显示明确错误  
✅ **批量导入**：支持Excel/CSV批量上传订单号数据  
✅ **统计监控**：实时查看订单使用情况统计  

## 技术架构

- **前端**：原生HTML + CSS + JavaScript（响应式设计）
- **后端**：腾讯云函数（Node.js）
- **数据库**：腾讯云开发数据库
- **部署**：腾讯云开发静态网站托管
- **成本**：免费额度足够日活1000+用户

## 快速开始

### 1. 安装依赖

```bash
npm install
cd cloudfunctions/verify-order && npm install && cd ../..
```

### 2. 部署到云开发

```bash
# 配置云开发环境（首次使用）
tcb login --key
# 请替换为您自己的 SecretId 和 SecretKey
# SecretId: YOUR_SECRET_ID
# SecretKey: YOUR_SECRET_KEY

# 创建环境
tcb env:create xiaohongshu-verify-prod -e shanghai

# 部署云函数
tcb fn:deploy verify-order

# 部署静态网站
tcb hosting:deploy .
```

### 3. 创建数据库

在云开发控制台手动创建集合：
- 集合名称：`xiaohongshu_orders`
- 安全规则：禁止前端直接读写

详细步骤见《部署指南.md》

## 使用说明

### 验证页面使用

1. 用户访问验证链接（自动嵌入企微回复）
2. 输入小红书订单号（格式：P + 18位数字）
3. 点击"立即验证"
4. 验证成功自动跳转到优惠券页面
5. 验证失败显示错误提示

1. 访问管理后台链接
2. 选择Excel/CSV文件上传
3. 预览数据（显示前10条）
4. 确认导入
5. 查看导入进度和结果
6. 查看统计信息（总数/已用/剩余）

| ...               |

**要求**：
- 第一行为表头（订单号ID）
- 从第二行开始是订单号数据

## 技术细节

### 验证流程

1. 用户输入订单号 → 前端格式验证
2. 调用云函数 → 查询数据库
3. 订单存在 → 删除订单号（核销）
4. 返回跳转链接 → 前端自动跳转
5. 订单不存在 → 返回错误信息

### 数据库设计

**集合**：`xiaohongshu_orders`

```json
{
  "_id": "自动生成",
  "order_id": "P790393366861231511",
  "status": "pending",
  "created_at": "2026-03-30T10:00:00.000Z",
  "used_at": "2026-03-30T11:00:00.000Z"
}
```

### 性能优化

- 数据库索引：`order_id`（唯一索引）
- 批量导入：每批100条，防止超时
- 前端防抖：防止重复提交
- CDN加速：静态资源全球分发

## 安全设计

1. **数据库安全**：仅云函数可读写，前端无权限
2. **格式验证**：前后端双重验证订单号格式
3. **防重复提交**：按钮禁用 + 加载状态
4. **事务保证**：查询后删除，确保数据一致性
5. **环境隔离**：生产环境独立，数据隔离
